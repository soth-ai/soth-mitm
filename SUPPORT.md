# Support

## Where to Ask for Help

- Usage and integration questions: open a GitHub issue.
- Bug reports: use the bug issue template.
- Feature requests: use the feature issue template.

## Before Opening an Issue

- Confirm you are on current `main` or latest release.
- Include exact command, config, and repro steps.
- Include relevant logs and failing lane artifacts from `artifacts/`.

## Not for Public Issues

- Security vulnerabilities: follow `SECURITY.md`.
- Secrets, credentials, or customer data: do not post publicly.
